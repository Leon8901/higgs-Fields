Context: how the real thing works (for reference while building)

Marketing Studio's actual mechanism: URL or manual product entry → LLM extracts structured product data → user picks one of four creative modes (each restructuring the whole generation approach, not just adding a style word) → optional saved avatar for spokesperson consistency → video model generates with native audio/lipsync in one pass → result saved to a persistent history.

Backend stack it actually runs on: a video model with native audio+lipsync generation as the core engine, a persistent identity/avatar system trained from multiple reference photos, and a two-tier LLM planning layer (cheap/fast model for structured tasks, stronger model for interpreting scraped product pages). Our build already has the OpenRouter planning layer and WaveSpeedAI as the generation gateway — this rebuild is about closing the structural gaps, not switching providers.


Phase 1 — Real creative mode switcher (do this first, no new generations required)

Replace the current UGC/Hook/Setting toggles with four distinct modes, each driving a genuinely different system prompt sent to the LLM planning layer:


UGC — face-to-camera, selfie framing, natural speech, iPhone-shot feel. Sub-styles: unboxing, tutorial/how-to, hands-on demo.
Professional/CGI — studio-grade, no presenter necessarily required, precise camera movement, product-reveal framing.
Cinematic — real-world locations, story arc, broadcast-commercial feel.
Wild Card — minimal structure, hands full creative direction to the LLM planning layer with a loose brief.


Each mode should map to a distinct system-prompt template (stored server-side, not hardcoded inline) that shapes pacing, camera language, and structure instructions differently — not the same template with one style word swapped in. Build the mode selector UI as a clear 4-way switch, not toggles that stack.

No new generations needed to build this — confirm before running any test.

Phase 2 — Persistent avatar system (reuses existing infrastructure)

Build a saved Avatar entity using the SAME identity-lock/BYOK infrastructure already built for the main generation flow — do not build a parallel system.


Users can create a named avatar from 1+ reference photos (start with single-image support; multi-image reference sets can come later).
Saved avatars appear in a picker on the Marketing Studio generation bar — select once, reuse across every future generation, instead of re-uploading each time.
Basic management: rename, delete, view which generations used a given avatar.


Phase 3 — Persistent product profiles

Build a saved Product entity: name, description, up to 5 images, created once and reusable across future generations — not re-entered every time.


Product profiles editable after creation.
URL-scrape flow should offer to save the extracted result as a reusable Product rather than a one-off.
Manual product entry flow should also save as a reusable Product by default.
